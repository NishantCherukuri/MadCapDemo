Risk and Threats Settings

The following risk and threats settings are available:

Risk Formula Customization

On this page, the user can fine-tune risk customization, set risk calculation host filters and time window, increasing or reducing the amount of time in which alerts contribute to the current risk score. Host filters can be set for:

IP addressProvides the option to include all hosts, private IPs only, or custom address ranges.
Last seenEnables to restrict the calculation to hosts seen in the selected time period (e.g. last 1 hour, last 24 hours, last 7 days, etc.).
Monitored networksEnables to restrict the calculation to hosts in the selected monitored networks.
Monitoring sensorsEnables to restrict the calculation to hosts from the selected sensors.

Security and operational risk scores are calculated taking into consideration a number of risk likelihood and impact variables. Each variable has a relative percentile weight, which totals to 100% for the calculation. These variables provide a quantitative way to calculate the risk automatically, based on the data available in eyeInspect.

The weights provide a way to fine-tune the calculation based on the preferences and priorities of the organization. The total percentage for each calculation must be 100% before changes can be committed.

After changing the weights to suit the organization's risk profile, the user can commit the changes by clicking Finish in the page sub-menu. To cancel the changes, click Back. After committing the changes, a new calculation is started immediately.

CVEs and IoCs - Update and Scan

The CVEs and IoCs page provides two main functionalities:
  • Enables the import of new asset vulnerabilities (CVEs) and threat intelligence (network and file Indicators of Compromise, or IoCs) from a single location.
  • Enables users to automatically scan all network logs to identify whether any threat is already active in the network, based on the latest threat intelligence (Forensic Time Machine).

Being up to date with the latest threat intelligence, and in particular with the latest CVEs and IoCs, ensures that eyeInspect is able to provide the most accurate possible asset vulnerability information and detect the latest security threats. Updates to the CVEs and IoCs database can be imported as a zip file by clicking Import in the sub-menu bar. Database updates are distributed monthly by the Forescout OT team, but can also be collected by the user from public sources or internal teams.

Database updates may contain the following threat intelligence:
  • Asset vulnerabilities (CVEs, distributed as XML files)
  • Blacklisted domain names (CSV or JSON file)
  • Blacklisted files or file operations (CSV or JSON file)
  • Blacklisted IP addresses (CSV or JSON file)
  • Blacklisted SSL client applications (CSV or JSON file)
  • Malicious file hashes (CSV or JSON file)
  • YARA rules (yar or yara files)

The import format for CSV files containing new IoCs of a given type is the same as the format of the exported CSV file for that IoC type. A CSV file containing the current IoCs of a given type can exported using the CSV Export functionality on top of the table of that IoC from the ITL page. Similarly, the format of the JSON file is the same as the format specified on the REST API documentation. The file name of the CSV or JSON file containing new IoCs of a given type must correspond to the event type ID of the ITL check corresponding to that IoC (for example, for new blacklisted IP addresses, the file name must be itl sec udb dns bip).

The Forensic Time Machine functionality automates the analysis of current and past network logs to identify malicious activity in the network, based on newly available threat intelligence. Often, the IoCs associated with a malware campaign are discovered only months after the initial infection. The Forensic Time Machine takes new IoCs imported into eyeInspect and verifies whether any of those was observed in the network in the last months. A new scan can be initiated either directly from the Import form of a new CVEs and IoCs database, or by clicking Network logs scan in the secondary nav bar. The result of a scan is presented on the right hand side of the page in the Network logs scan results table. If any IoC was positively encountered among the network logs during a scan, users can click on the corresponding take entry to get more details, export the results and initiate response.
