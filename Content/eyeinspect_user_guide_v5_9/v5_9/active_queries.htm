Active Queries

The Active Queries page shows a list of active queries and associated details in a tabular format. Click a row to open the Scan details window to view detailed information about the query.

To perform actions on one or more active queries, select the respective checkbox(es) in the first column to activate the action icons above the table -

The actions are, in order:

  • Download the query details in a zip format
  • Stop the active query
  • Remove the active query
  • Reload the active queries table

Start an Active Query

Click Start an Active Query from the secondary navigation bar to open the Start an Active Query dialog, which contains the following steps:

Step 1: Configuration

Select the query type and the associated parameters that define how the active query will be executed by the sensor.

Step 2: Target

Select the target devices either by choosing from predefined filters or by applying filters based on IP Reuse Domains and Network Addresses. Note that IPv6 network addresses are supported.

Step 3: Options

Choose the query options from the following:

  • Active Query: The scan is executed immediately upon completion of the flow.
  • Active Query Plan: The configuration is saved and can be scheduled for future execution via the Active Query Plans page.

If you are unsure about specific configuration values, you can save the setup as a query plan and edit it later using the same dialog. Note that once saved as a query plan, the type cannot be changed - it will remain a query plan.

Step 4: Summary

This step presents a consolidated view of the configuration defined in the previous steps:

  • Sensitive information, such as passwords, will not be displayed in this summary.
  • Depending on the selected option in the previous step, the user will be presented with either a Start button (to execute an active query) or a Save button (to store the query plan).
  • If any validation errors are detected based on the configuration, they will be clearly highlighted, allowing the user to make necessary corrections.

Upon successful completion of the form, a confirmation message is displayed.

Improve OS/Ports and OT Ports Query Performance

For active queries of OS/Ports and OT Ports query types, the following two new parameters are introduced to improve the query performance:

Parameter Value
OVERWRITE_SERVICE_DISCOVERY Accepts boolean value: True/False
OVERWRITE_AGGRESSIVENESS Defines the aggressiveness level on a scale of 1 (least) to 5 (most).

To use these parameters, do the following:

  1. Log in to the Active sensor host using SSH.

  2. Edit the /opt/active-sensor/.env file and add the parameters. For example:

    OVERWRITE_SERVICE_DISCOVERY=True
OVERWRITE_AGGRESSIVENESS=3

  3. Save the file.

  4. Restart the container by running the following commands:

    sudo /opt/active-sensor/manage-containers.sh --rm
sudo /opt/active-sensor/manage-containers.sh --up
