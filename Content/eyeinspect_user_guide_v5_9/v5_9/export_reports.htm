Export Reports

The eyeInspect Command Center allows users to export and schedule reports in .DOCX and .CSV formats.

Two types of reports are available:

  • Analysis - This option exports an analysis of host and alert information in .DOCX format
  • Raw data - This option exports one or multiple .CSV files that contain the plain data of the asset inventory and alerts

The export report window contains the following steps:

Step 1: Information

The first step is to enter a report name, select the report type and to select the information to be included in the report. The following information can be included, based on the report type selected:

Report type: Analysis (Word Doc.):

  • Overview - This page shows an overview of your network in terms of asset classification, protocols and alerts.
  • Asset Report - Includes Asset classification (Purdue Model) and Vulnerable assets
    • (Optional) Include Detailed Asset Inventory information, with a limit of 500 assets
  • Alerts Report - Includes Security, Operational and Networking alerts. Alerts are grouped by Event type ID, Severity, Event specific info, L2, L3, L4, L7 Protocol, Sensor, Engine Type and Profile/Module.
    • (Optional) Include Non-aggregated alerts with a limit of 2000 alerts
    Note: The Command Center may become unresponsive when exporting a report of type Analysis (Word Doc) with a large number of devices. To avoid this issue, it is recommended to export a maximum of:
    • Assets view: 110k entries
    • Devices view: 110k entries
    • Links view: 640k entries
    • Alerts table view (aggregated): 1.5 million entries
    • Alerts table view (non-aggregated): 220k entries

Report type: Raw Data (CSV):

  • Asset Inventory - Include the asset inventory information you want in the report. Choose from Assets, Changelogs, Links and Vulnerabilities. Multiple options can be selected.
  • Events - Include the event information you want in the report. Choose from grouped alerts and ungrouped alerts. Both options can be selected.

Step 2: Source

Select the sources for the Asset and Alert Tabs to get information to populate the report from. It is possible to choose the default source, or select from the list of templates.

Note: Users will only be able to see the sources which they have permissions for.

Note: The list of available sources depends on the selections made in the Information step:
  • If Overview report type is selected, there are no sources necessary and hence the sources will be greyed out.
  • If Asset or Alerts report type is selected, the Asset or Alert Tab sources will be available respectively.
  • If both Asset and Alerts report types are selected, both the Asset Tab and Alert Tab sources will be available.

Step 3: (Optional) Schedule

From the Schedule tab, it is possible to automatically email the reports on a periodic basis. Scheduling reports is optional. The email server and the recurrence details can be set on this tab, along with the list of recipients, the email subject and an optional message.

Step 4: Summary

The summary tab shows the summary of the options selected in the previous tabs.

Once the report is exported, a popup is displayed over the top navigation bar with the status of the report.

Note: For optimal performance, CSV file downloads have the following limits:
  • 100K entries for assets/changelogs/vulnerabilities
  • 600K entries for links
  • 200K entries for alerts

Note: Do not navigate away from the Export Reports window until the export is complete.

Manage Scheduled Reports

To manage scheduled reports, navigate to Settings > System Settings and from the Reports section, select Manage Reports. Scheduled reports can be paused/resumed, edited, and deleted. After editing a report, a download will be triggered.

Note: Users can only export or schedule exports for data they have permission to access (Monitored Hosts or Alerts). Otherwise, an “insufficient user permissions” message is displayed.

Filters for a Scheduled Report

  • A scheduled report uses the filters of an asset/alert tab as they were set at the moment of creating the schedule.
  • If the user changes/deletes the tab, the filters that were used to generate the scheduled report will not be affected.
  • When editing a report, the user may select a new tab for the assets or alerts section to update the filters.

Required Permissions

  • There must be a Mail Server configured specifically for the export doc report feature to schedule reports. Only users with edit permissions for the Mail Server Configuration resource may setup/edit a Mail Server.
  • Users without view permissions for the Mail Server Configuration resource won’t be able to schedule a report, only download it immediately.
  • Users with view permissions on Alerts and Monitored Networks will be able to view and edit the scheduled reports of other users. Users missing one of these permissions will only be able to see their own scheduled reports.
