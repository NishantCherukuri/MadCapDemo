User Settings

 

A User account is required to access the eyeInspect Command Center. Access control in eyeInspect is achieved through rules, roles and users, where rules define authorizations to access specific functionalities and resources. A role is assigned a set of rules and users can be mapped to one or more roles. Consider for example a user "John", who has the role of "Human Resources", which is assigned authorizations (rules) that allow John to Add, Edit and Remove Users from the Command Center.

The following user settings are available:

Users and roles

Users:

The eyeInspect Command Center includes one predefined user: admin, which has admin as default role. To view the current configurations and add a new user go to "Settings" > "System Settings" > "Users and roles".

Users that are locally authenticated by the Command Center show as Source "internal", while externally authenticated users, e.g., using LDAP, are marked "external". Assigning a role to a User grants privileges on eyeInspect resources as defined by that role. A User can have more than one role.

Roles:

The Command Center includes four predefined roles:

  • admin - The eyeInspect system administrator, authorized to access and modify every configuration of the platform
  • analyst -The eyeInspect analyst, mostly responsible to analyze the network, its assets and alerts, and refine the detection settings of Sensors
  • viewer - A role that grants read-only access to the network, assets and alerts information, as well as system health. Viewer users have editing (but not deletion) rights only on alerts and alert cases
  • blank - Role reserved for LDAP integration and authentication

Note: A new role does not have any rules associated with it.
Note: It is not possible to delete a role that is currently associated with one or more Users. These Users have to be deleted or associated with a different role, before the role can be deleted.

LDAP

LDAP authentication can be enabled and configured to authenticate external users to the eyeInspect Command Center using an LDAP directory server. Users connecting with LDAP authentication are listed on the Users page with the source attribute set to External. LDAP authentication can be enabled via "Settings" > "System Settings" > "LDAP". For more information about the settings hover over the question mark in the text box to see an explanation of the setting.

User activity log

The user activity log keeps track of all Users' activities executed both on the Command Center user interface and on the eyeInspect servers. Logged actions include, but are not limited to logging in and out; adding, removing and editing of resources such as sensors, filters, profiles and roles; inserting/removing USB sticks; and executing elevated commands. For each activity the date, time, username, IP, affected resources and what action was performed is recorded in the log. Log entries cannot be deleted, not even with the "admin" account. This information is not stored as asset information. It is only logged as an event.

User activity log forwarding

If remote logging of user activities is enabled, the user activity logs will be forwarded to a remote syslog server. This section describes the procedure to forward user activity logs.

1. Navigate to Settings > System Settings > Users > User activity log forwarding

2. Select the Enable forwarding checkbox.

Select/enter the following forwarding settings:

Connectivity settings

Option Action
Transport protocol Choose UDP
Remote host The IP address of your server
Remote port The port number your receiver is listening to.
Click TEST CONNECTIVITY to test successful connection.

Header settings:

To enable header settings, navigate to the Header tab and select the Send message header checkbox.

The severity can be chosen in the Header settings. The default value is ALERT.

It is possible to set a message to be sent with the user activity log. Select Use pre-set message from the top menu to enter a pre-set message. The following pre-set message options are available:
