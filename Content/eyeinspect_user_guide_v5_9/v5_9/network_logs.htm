Network Logs

Aside from Alerts, Sensors also monitor the network for other interesting events such as remote authentication, DNS name resolution, and file operations. These events are known as Network Logs and they are logged by the Event Logging Module and sent to the Command Center. The Command Center aggregates them and displays them in the Network Logs page.

The Network Logs page provides a quick overview into how often logged events occur in the networks and provides default views on various Event types, as well as the option to customize the page to show the details of interest to the user. By clicking "IoC scan page" in the sub menu bar, users can access the CVEs and IoCs page, where they can verify whether newly available IoCs can be identified among current and past logs.

SD Scripts can receive a callback whenever certain Network Logs are detected. Currently the following Network Logs are supported in SD Scripts:

  • Name query (DNS)
  • Name resolved (DNS)
  • File access error (FTP, SMB, SPLUS)
  • File create (FTP, SMB)
  • File delete (FTP, SMB)
  • File read (EtherCAT, EtherCAT Routable, FTP, SMB, SPLUS, TFTP)
  • File rename (FTP, SMB)
  • File write (EtherCAT, EtherCAT Routable, FTP, SMB, SPLUS, TFTP)
  • Failed authentication (DCOM, ExperionHMI, ExperionPointBuilder, FTP, HTTP, Kerberos, LDAP, SMB, SNMP, SPLUS, TELNET)
  • Successful authentication (DCOM, ExperionHMI, ExperionPointBuilder, FTP, HTTP, Kerberos, LDAP, SMB, SNMP, SPLUS, TELNET)
  • SSL certificate (SSL)

Note: Failed and successful remote authentication attempts are logged for SNMPv1, SNMPv2, and SNMPv3. SNMPv3 authentication is encrypted, so eyeInspect can only extract user details for SNMPv1 and SNMPv2.

The secondary navigation bar contains the following options:

Reload

Reloads the network logs table.

Settings

Edit the table settings.

CSV Export

Export the network logs in CSV format.

Network logs scan

Opens the network logs scan page.
