Maintenance Settings

 

The following maintenance settings are available:

CC maintenance mode

Some of the database maintenance activities required to guarantee the correct and performance functioning of the Command Center are slowed down or even blocked when the system is under a heavy transaction processing. Sensors are the main responsible for transaction activities in the Command Center database.

The CC maintenance mode functionality can temporarily set the Command Center in maintenance mode (i.e. pause and resume Sensors' transactions) to ensure that maintenance activities can successfully complete. The CC maintenance mode is automatically set when database maintenance activities are started, but can also be manually set by the user (e.g. to enable a faster deletion of a very large number of Alerts or hosts).

The Command Center can be in one of the following three maintenance modes:

  • Regular - No maintenance ongoing
  • Automated maintenance - Maintenance mode was activated by a database maintenance activity (e.g. vacuuming, re-indexing)
  • Manual maintenance - Maintenance mode was activated by the user
The user can enable or disable manual maintenance mode only when the Command Center is in regular mode, whereas automated maintenance mode will be activated both if the Command Center is in regular or manual maintenance mode. During automated maintenance mode, users will be prevented from performing operations with large impact on the database (e.g. bulk delete of a large number of Alerts).

Users can enable manual maintenance mode by clicking on "CC maintenance mode", and then on the "Enable" button; a yellow bar will appear under the Command Center sub-menu to inform the user that the Command Center is under maintenance. Users can at any moment disable manual maintenance by clicking on "Disable maintenance".

Activity scheduling

Certain maintenance operations, such as re-indexing the database tables or deleting old Alerts, can be scheduled to execute at predefined times. To add a new scheduled activity, click the + icon above the scheduled activities table. The following activity types are available:

Database Maintainer - This job executes maintenance operations on the Command Center database to optimize the performance and resource usage. The options are the following:
VacuumReclaims "lost" storage space due to deleted Alerts. Note that this is a heavy operation and should not run more than once per month. A re-index is performed as part of the vacuum operation.
AnalyzeBuilds statistics of the database table contents. The PostgreSQL query planner uses this information to optimize queries.
ReindexRebuilds the database table indexes which are used in search operations.

Note: Vacuum and re-index operations must not be scheduled at the same time. Only use re-index if there is no vacuum required. Otherwise, use vacuum.

Alert Cleanup - The job deletes old alerts from the database to avoid running out of disk space in the Command Center.

Host Change Log Cleanup - This job deletes old Host Change Log events from the database to avoid running out of disk space in the Command Center.

Risk Calculation - This job updates the risk scores of all the Assets in the database.

Asset Analytics Refresh - This job updates the Asset data source used in analytics. By default, a refresh is done every 60 minutes.

Asset Deviations Computation - This job computes deviations of host properties from Asset Baseline policy constraints.

Host Cleanup - This job deletes hosts which have not been seen for the specified amount of days.

Default schedules - Default schedules will be automatically added for new deployments. For existing deployments, in case one of the above activities is not scheduled, the eyeInspect Command Center will add a scheduled activity using the default values listed in the below table. When changing activity schedules, please ensure there are no overlapping schedules to avoid unnecessary strain to the system.

Clear data

The Clear data page allows you to clear the sensor, Command Center and user settings and logs data.

Only users with the admin role can execute this operation and will need to enter their password.

The Clear data page displays a table of the available analytics datasources and details such as the first and last segment date, size, status, date and result of the last cleanup.

Note:
  • If you encounter an error when trying to clear data, confirm that the datasources have actually been removed, and try to clear data again.
  • No user other than the one executing the operation can be logged in to the Command Center when the clear data operation is executed. A warning will be issued if this is not the case when attempting the operation.
  • Only one cleanup operation can be active at the same time. Wait for the cleanup operation to complete before starting a new operation.
  • The cleanup operation should be used with care. A cleanup does not need to be run to optimize the performance and resource usage of the analytics database. The database automatically performs optimizations.

Software and license management

The Software and License management page allows easy management of the licenses of all sensors connected to the Command Center. The page shows the software version, license level, expiration dates and protocol restrictions. New licenses can be uploaded and sent to one sensor or multiple sensors at once.

Outbound Certificates Management

The outbound certificates management page allows users to manage certificates for outbound services such as: syslog forwarding, email forwarding and LDAP configuration.

Storing rules and notifications

The storing rules and notification settings provide for asset storing, which refines how eyeInspect deals with hosts.

The asset storing rules allow users to prevent assets from being stored in the Command Center's database and displayed in the Network Map and Monitored Hosts list. This is useful, for example, for Sensors monitoring networks communicating with external hosts which are out of scope and responsibility of the organization. Storing filters can be set for:

  • IP address - Provides the option to include all hosts, private IPs only, or custom address ranges.
  • Monitored networks - Provides the option to include all hosts, private IPs only, or custom address ranges.

Health status log forwarding

When enabling health status log forwarding, health status changes above or below the warning/critical level will be forwarded to a remote syslog server, so that the user can be warned about any issues while not actively using the eyeInspect Command Center. Remote health logging can be enabled via "Settings" > "System Settings" > "Health status log forwarding".

The configuration is the same as user activity log forwarding, the only difference lies in the available tags.
