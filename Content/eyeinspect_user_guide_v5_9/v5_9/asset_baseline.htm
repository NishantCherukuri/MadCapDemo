Asset Baseline

The Asset Baseline page displays a list of hosts in the asset baseline in a tabular form. In addition, you can perform the following Asset Compliance checks on this page:

  • Assign and un-assign hosts to Asset Baseline (compliance) policies.
  • View the current compliance status of each host and access a dedicated policy compliance page.
  • View the list of existing Asset Baseline policies and their details.
  • Export the list of existing policies and current compliance deviations.

Asset Baseline Policies

In order to assign one or more hosts to a compliance policy, make sure that the "Asset baseline" column filter includes "Unassigned" hosts; additional host filters can be applied from the table columns and the "Filters" menu. Then select the desired hosts from the table and click on "Add to asset baseline". This will open a modal form from which the user can choose whether to assign the hosts to an existing or new policy. When creating a new policy, two policy types are available for selection: Group policy and Individual policy. Group policies are defined in terms of constraints that all hosts assigned to the policy must comply with. Constraints can be defined for the following host properties:

OS version

The version of the Operating System installed on the host (e.g. Windows 7). Constraints can be defined using the "equals" or "contains" matching type. If "equals" is used, a host is compliant only if its ”OS version" property is an exact match with the string specified in the constraint; using "contains", on the other hand, allows users to specify a sub-string of the observed ”OS version". For example, consider a host with OS version "Windows Server 2008 R2”, and a policy constraint with value "Windows Server 2008”. The host is compliant with the policy if the constraint is specified as "contains", and non-compliant in case "equals" is used.

Open ports

The ports identified as open by an Active Sensor scan. Constraints can be specified for both TCP and UDP ports using matching type "equals" or "allowed". Matching type "equals" requires an exact match with the observed property value for a host to be considered compliant, whereas "allowed" defines a superset of permitted open ports. For example, consider a policy constraint defining TCP ports 80 and 443 as "allowed"; a host whose only open port is port 80 is compliant with this policy; a host with open ports 80 and 502 is not compliant. Note that these constraints are only applied to the "Open ports" property populated by active queries; it does not apply to the "Server protocol" property populated by Passive Sensors.

Firmware version

The version of the firmware installed on the host. Constraints can be defined using the "equals" or "contains" matching type, similar to OS Version.

Individual policies use the current values of the host's "OS version", "Open ports" and "Firmware version" properties as constraints. In other words, individual policies allow users to set the current host configuration as baseline, and report any configuration change as a compliance deviation. An overview of the existing Group policies and their details can be accessed from the Asset Baseline page by clicking on "Group policy list".

Policy and Compliance Details

The Command Center computes Asset Baseline compliance deviations at regular intervals. The "Last compliance check" timestamp above the Asset Baseline table shows when the last deviation was computed. Users can quickly identify non-compliant hosts by sorting or filtering the table using the "Compliance status" column. A red exclamation mark followed by a number in parenthesis indicates how many of the host's properties are deviating from the policy constraints. Compliance policies and deviations can be exported by clicking on "CSV Export". Clicking on each table row opens the Asset policy details page, which presents the compliance status and deviation details of a host. This page provides a preview of the host (Asset identification summary panel), of the policy (Policy Info panel), and lists the expected (by constraint) and actual property values of the host. Non-compliant properties are marked with a red exclamation mark so that they can be easily identified. The host can be unassigned from the policy by clicking "Remove from asset baseline". Host and policy compliance status can also be viewed from the Group policies list page. Users can quickly determine which of the existing policies contains non-compliant hosts by sorting or filtering the table using the "Compliance status" column. Details about non-compliant hosts can be accessed from this page by clicking on one of the policies in the table. This will open the Policy details page, from which users can also edit the policy and its constraints.

The secondary navigation bar contains the following options:

Group policy list

Displays a list of all group policies in tabular form.

CSV Export

Exports the list of asset baseline hosts in CSV format.

Filters

Displays the full list of filters that can be applied to the hosts.
