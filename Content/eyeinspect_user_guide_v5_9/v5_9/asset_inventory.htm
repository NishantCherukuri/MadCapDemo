Asset Inventory

The As

set Inventory page offers customizable dashboards that provide rich situation awareness of the current and past network activities. Widgets and tabs can be filtered to show specific protocols or networks. The Map provides a rich visual overview of the hosts and their communications as detected by the sensors connected to an eyeInspect Command Center, as well as functionality to filter hosts and see potential issues. The Asset Inventory also allows users to create tabs in order to persist customized views on the various asset tabs, such as Map, Assets, Devices, Links, etc.

The Asset Inventory page contains a number of tabs in the top half, and various tables in the bottom half, both providing visual information about the various assets connected to the Command Center.

Asset Tabs

Each asset tab contains a view of the Assets with the hosts and networks as seen by the Sensors connected to the Command Center. Every time a new host is seen by a Sensor, it will automatically be added to the Assets Map. Select a host from the map and choose from Asset Details or Communication Details to view more information.

Asset Table Templates:

Upon selecting the Add a New Tab button , the user can choose from a list of predefined templates. The currently available Asset templates are:

  • Default

  • Active Assets

  • All Assets

  • All Vulnerabilities

  • CISA-KEV

  • Comms with Public IPs

  • High & Critical Risk
  • High Confidence Vulnerabilities

A user can rename, clone, export and delete tabs to create a view in which all information relevant to them is easily accessible. For each selected Asset tab, click the three dots next to the tab name to view the following options:

  • Rename: rename the tab
  • Clone: clone the tab
  • Export: export the tab in JSON format
  • Delete: delete the tab

In each asset tab, the following charts are displayed:

  • Assets per risk
  • Vulnerability instances per severity
  • Assets per purdue level
  • Top protocols

Asset Views

With each Asset tab, the following views are available in the bottom half of the screen:

Map

Displays the assets in a map view. Several display options are available for the map. Click the three dots next to Maps to view the map options. The following preferences can be set:

  • Layout - Two layout options are available via the "Layout" option. The default option is to sort the hosts and networks by "Purdue level". A "Communication View" layout is also available, which tries to show all the hosts and networks while maximizing the amount of space available. The left hand side of this layout shows a vertical bar ranging from Purdue level 0 to 5. The grouping is maintained in both layouts.
  • Group Assets by - By default, the Assets Map is grouped by host role. Alternatively, the Assets Map can be grouped by monitored network to have hosts grouped by the network to which they belong.
  • Cluster Nodes - When multiple hosts of the same type perform the same network communications, these are clustered together. To force all hosts to be displayed separately, un-check the option "Clustering Enabled". When combined with the "Display host names", this provides an operator with a stronger visual queue to recognize issues in IP-range assignment.
  • Display - The following three display options are available:
    • By default, the map displays the host name of a host if it is known, and the IP or MAC otherwise. To show IP or MAC addresses even if the host name is known, toggle the "Hostname" switch. This behavior is useful to visually see discrepancies in the assigned IP ranges, such as gaps or out-of-range IPs.
    • Toggle the Assets Communicating with Filtered Assets switch to view these assets.

Actions:

Navigate to the Actions menu above the Map view and click the downward-facing arrow to open the dropdown menu. The following options are available in the Actions menu:

  • Map > Export as .PNG: Exports a PNG image of the current map view (including selection, zoom, etc) for offline analysis or reporting.
  • Assets > Create Missing Networks: If there are any hosts in the map that do not fall within any monitored network, create this network using this option. This will display a form in which the CIDR prefix for IPv4 and/or IPv6 can be specified. Monitored networks can also be manually added or edited from the monitored networks page.

Assets

Displays all connected assets in a tabular format with the following default columns:

  • IP Address
  • Hostname
  • Role
  • Security Risk
  • Operational Risk
  • MAC Vendor
  • Vendor and Model
  • OS Version
  • Firmware Version
  • Server Port

To add more columns, navigate to the Columns menu above the Assets table and click the downward-facing arrow to open the dropdown menu and select/deselect columns.

To view asset details, click any value in the corresponding row. This opens the Details window.

Actions:

To view the available asset actions, select one or more assets by checking the checkbox in the first column, and navigate to the Actions menu above the Assets table. The following actions are available:

  • Create Missing Networks - as described in the Maps section
  • Edit Assets
  • Unset Assets - Unset releases the manually forced values and restores the default priority for the identified information sources to the sensor(s). The value will update when a change is detected by the sensor
  • Delete Assets

Devices

Displays all connected devices in a tabular format with the following default columns:

  • IP Address
  • Hostname
  • Role
  • Security Risk
  • Operational Risk
  • MAC Vendor
  • Vendor and Model
  • OS Version
  • Firmware Version
  • Server Port

To add more columns, navigate to the Columns menu above the Assets table and click the downward-facing arrow to open the dropdown menu and select/deselect columns.

To view asset details, click any value in the corresponding row. This opens the Details window.

From the Details window, it is possible to group/ungroup assets to the selected Device. To do this, go to the Asset Properties tab and click the three-dot menu next to the Choose Asset from dropdown, and click Group more Assets or Ungroup existing Assets.

Links

Each row of the links table represents aggregated links based on source and destination. All the information that are different among those links (like protocols) is represented as a list. It is by default sorted by the column "Last Seen". The Links table has a switch called "Include links between Filtered Assets and other Communicating Assets" which is off by default. Switching it on will show links not only between the filtered assets but also include the assets communicating with any of the filtered assets.

The following columns are displayed by default:

  • Source IP Address
  • Destination IP Address
  • Source Role
  • Destination Role
  • Protocols (Ports)
  • First Seen
  • Last Seen
  • Bytes Sent
  • Bytes Received

To add more columns, navigate to the Columns menu above the Assets table and click the downward-facing arrow to open the dropdown menu and select/deselect columns.

Vulnerabilities

Displays a list of CVEs (Common Vulnerabilities and Exposures) in a tabular format with the following default columns:

  • CVE ID - Click the downward arrow next to the CVE ID to expand the row with a description of the CVE, along with the solution and references.
  • Exploitability Tags
  • Title
  • CVSS Score
  • # of Affected Assets
  • Affected Assets
  • Suppressed for
  • Remediation Level
  • Vendor
  • Last Modified

To add more columns, navigate to the Columns menu above the Assets table and click the downward-facing arrow to open the dropdown menu and select/deselect columns.

Actions:

To view the available CVE actions, select a CVE by checking the radio button in the first column, and navigate to the Actions menu above the Assets table. The following actions are available:

  • Suppress
  • Unsuppress

Changelogs

Displays a list of changes to assets in a tabular format, with the following default columns:

  • Timestamp
  • Event Type
  • IP Address
  • Hostname
  • Sensor
  • Old Value
  • New Value

To add more columns, navigate to the Columns menu above the Assets table and click the downward-facing arrow to open the dropdown menu and select/deselect columns.

Software

Displays a list of software in a tabular format, with the following default columns:

  • Retrieved On
  • Name
  • Vendor
  • Version
  • Installed on Assets

The data for this table is retrieved from the Active Scan (WMI).

Note: All the above Asset table views, except Map view, can be exported as a CSV by clicking the three dot menu next to the view name and selecting Export CSV.

Filters

Users can apply filters to all the above charts and tables. The Filters row is present above the table.

Filtering cross-checks every host in the respective Asset view for a given condition. Only the hosts which match that condition are displayed. All filter fields that are filled in will be combined with a logical AND to create the condition to which a host must conform to in order for it to be visible in the table. The current filter(s) can be deleted by clicking Reset All Filters at the end of the Filters row.

Additional Features

The Map, Assets, Devices, Links and Changelogs tables provide three additional features that can be accessed by navigating to the Additional Features menu above the table and clicking the downward arrow.

The following additional features are available - click each feature name to know more.

  • Threat Scenarios (Classic Map)
  • Host Editing (Monitored Hosts)
  • Host changelogs

Custom Properties

To view, create, edit or delete custom properties, navigate to the secondary navigation bar on the Assets page and click Custom Properties.

A built-in custom property named Groups is available to enhance asset categorization and filtering across the eyeInspect Command Center and is integrated into the Custom Asset Property, Asset, and Alert pages.

The Groups custom property cannot be edited or deleted, accepts string values and allows multiple group entries per asset.
