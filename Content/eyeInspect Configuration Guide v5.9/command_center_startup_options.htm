Command Center Startup Options

The eyeInspect Command Center (CC) has a number of startup options, which are described below. To use the startup options, modify or create, if non-existent, /opt/sdconsole/conf/application.properties and use one of the following options in the format property=value. After saving your changes, start the Command Center by executing the following command: 

sudo supervisorctl start silentdefense-cc

If the Command Center was already running, restart it by executing the following command:

sudo supervisorctl restart silentdefense-cc

Additional configuration parameters can be found and edited in the files /opt/druid/conf/historical/runtime.properties and /opt/icsp/icsp-cc/conf/icsp.conf.

The parameters in file runtime.properties can be used to tune the Druid configuration to ensure the best data processing and retention strategy, based on the available system resources. The table below lists the most relevant ones. It is required to restart the Command Center for changes to apply.
OptionDefault ValueDescription
druid.segmentCache.locations[{"path": "/tmp/druid/indexCache", "maxSize": <25% of the disk size>}]The (maximum) disk space used for segments that are to be assigned to the Druid historical node. If 95% of this limit is reached, older segments will be deleted to ensure new data can be correctly processed and displayed (this threshold is configurable from the Command Center Diagnostics page).

The parameters in file icsp.conf can be used to tune the Active Sensor configuration, e.g. for log collection and retention. The table below lists the most relevant ones. It is required to restart the Command Center for changes to apply.
OptionDefault ValueDescription
rotate5The maximum number of log files that are stored. Older files will be deleted when new ones are created.
maxsize100The maximum size (in MB) of log files. Log files will be rotated if this limit is reached.
