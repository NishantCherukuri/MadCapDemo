Command Center High Availability

 

Note: Updates to the vulnerability database will not be synchronized between primary and secondary Command Center. If switching the Secondary Command Center to Active Mode is required, it is recommended to update the vulnerability database to the latest version provided on the Forescout OT Portal.

The eyeInspect Command Center is able to operate in a "warm standby" High Availability mode. This means it is possible to configure a second eyeInspect Command Center to act as a replica of a primary eyeInspect Command Center. The replica will synchronize its main database (assets, alerts and related information) and the analytical database with the primary server.

While an eyeInspect Command Center is in High Availability mode as secondary, the eyeInspect Command Center web interface is not accessible and the database is marked as read-only, to prevent any conflicts when synchronizing. The following sections describe how to configure the High Availability mode and how to switch the Secondary Command Center to Active.
