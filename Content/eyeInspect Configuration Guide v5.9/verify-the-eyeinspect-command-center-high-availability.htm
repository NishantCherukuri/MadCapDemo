Verify the eyeInspect Command Center High Availability

 

The following steps will verify that the synchronization works on both Command Centers (Primary and Secondary).
  1. In this procedure the database synchronization will be verified by executing a PostgreSQL query on the Secondary Command Center and checking the ”user activity log” value on the Primary Command Center
    1. Login into the eyeInspect Primary Command Center via web-browser
    2. Check the number of user activity logs by selecting Settings , then User activity log . The uppermost entry should show a log regarding the login done in Step (a)
    3. On the bottom left of the page you will see the total amount of user activity logs on the Primary Command Center
    4. SSH into the secondary Command Center and execute the following command. The result will show the total amount of user activity logs on the Secondary Command Center 
      sudo -u postgres psql -d silentdefense -c "select count(*) from logs;"
    5. If Step (b) shows an entry about the last login and results from Step (c) and (d) are matching, the database is correctly synchronized.
  2. In this procedure the database synchronization will be verified by executing a PostgreSQL query on both Command Centers (Primary and Secondary) and checking the number of alerts on the Primary and Secondary Command Centers
    1. On the Primary Command Center connect to the PostgreSQL database and query the number of alerts by executing the following SQL command: 
      sudo -u postgres psql -d silentdefense -c "select count(*) from alert_details;"
    2. On the Secondary Command Center connect to the PostgreSQL database and query the number of alerts by executing the following SQL command: 
      sudo -u postgres psql -d silentdefense -c "select count(*) from alert_details;"
    3. If the number of alerts appearing in Step (a) and Step (b) are matching, the database is correctly synchronized.
