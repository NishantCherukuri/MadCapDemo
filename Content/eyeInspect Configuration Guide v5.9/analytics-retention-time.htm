Analytics Retention Time

The Druid database stores network events and flow information to enable real-time visualization and forensic analysis via the eyeInspect Dashboard, (Visual) Network Analytics widgets, and from the Network Logs page.

This option allows users to view and customize the retention time of the Druid analytical database.
The Druid database stores network events and flow information to enable real-time visualization and forensic analysis via the eyeInspect Dashboard, (Visual) Network Analytics widgets, and from the Network Logs page.
Note: The recommended retention time is 30 days; this is the optimal trade-off between functionality and system performance. However, a longer retention period may be required for compliance purposes. A longer retention time may decrease system performance, i.e. lead to slower loading times of analytics data and pages.
